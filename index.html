Banner image is jpeg format.

4. Ad shows GIF image

Banner image is gif format.

5. Ad shows PNG image

Banner image is png format.

6. Ad shows HTML

Generic HTML Banner would be injected into dashboard preview section directly without using iframe and it may break the layout.

Better use HTML5 Banner (ZIP package)

7-9. Video ads (pre-roll, mid-roll, post-roll)

It supports mp4 and webm format only.

It output link of VAST2 xml for video player integration. VAST2 does not control ad placement, can do it in video player.

The demo process both the VAST and the video at client side. Consider using Server-Side Ad Insertion (SSAI) to prevent ad blocker.

10. Ad targets specified browsers

It shows different banners for chrome, firefox and safari respectively.

11. Ad targets specified geolocation

It shows banner if the user is in HK.

ref: Read this before setup

12. Ad targets specified operating system

It shows respective banner when the user is using Mac, Windows, iOS or Android.

13. Display ad by different article ID

It reuse the same zone but different article IDs (through tag variable) to show respective banner.

Using "Async JS" tag type will always show the same banner as the tag values wil be overwrite each other for the page call.

Either use non-async-js tag type or place them in different pages.

ref: Tag Variables, Site - Variable
